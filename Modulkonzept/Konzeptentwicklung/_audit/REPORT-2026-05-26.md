## Executive Summary

1. Das laut `Mitarbeiter_Webapp.md` §3.3 zentrale **Task-Dashboard existiert nicht als echtes System**: keine `tasks`/`task_collaborators`/`task_activity_log`-Migrationen, keine Backend-Routen, keine `/tasks`-Route im Webapp-Router. Das Dashboard rendert eine **hartkodierte Projekt-Todo-Mock-Liste** (`webapp/src/data/tasks.ts`, eingebunden in `webapp/src/pages/DashboardPage.tsx:13`).
2. Größter unaufgelöster Drift: zwei **parallele Daten-Welten**. `01_Datenmodell_Events.md` §6 beschreibt `customers`/`customer_profiles`/`receipts`/`idempotency_keys`/`processed_events`; der reale Schema-Reboot (`backend/migrations/030_belege.sql:11`) erklärt `receipts` für tot und nutzt `tenants`/`belege` + RLS via `current_tenant_id()`. Trotzdem registriert `backend/src/app.ts:267–302` **beide Welten gleichzeitig** (`/customers`-Routen, Module `customers/`, `profiles/`, `documents/`, `reports/`, `receipts/` neben `/tenants` + `/belege`).
3. **Migrations-Referenzen in drei Specs falsch:** M14 nennt `031_users_auth.sql` (real `020_users_auth.sql`), M15 nennt `040/041_pos_*` (real `022_pos_credentials.sql` + `040_kasse.sql`, **kein** `pos_daily_close`), M13 nennt `028_tax_advisor_portal.sql`, **das gar nicht existiert**.
4. Pilot-kritisch fehlend bzw. ungebaut: Auto-Trigger-Engine (Webapp §5), Onboarding-Wizard, Discord-Bot/Notifications. Legitim Post-Pilot (laut Webapp §10): Auto-Rechnungs-Generator + `invoices` (§6), Provisions-Übersicht (§3.6), Customer-Chat-View (§3.5), Web-Chat-Widget.
5. **STATUS.html ist veraltet** (Titel 2026-05-07, Selbst-Banner „Veraltet seit 2026-05-12", `STATUS.html:6,151,155`), älter als die zuletzt geänderten `modules/`-Specs (2026-05-18); **M15 (SumUp) fehlt darin komplett**.
