5 · Offene Fragen — Klärungsbedarf

❓ #F1 — Soll m06-advisor-portal/ wirklich zu m13-advisor-portal/ umbenannt werden?

M14-Spec deutet es an, aber Code ist stabil und Tests grün. Umbenennung kostet 30 min + Risiko, dass irgendwo ein Import-Pfad zerschossen wird. Pro: konsistente Namensgebung. Contra: Touchen eines stabilen Bereichs für Kosmetik.

❓ #F2 — Soll der Initial-Bootstrap (031b) eigentlich noch in Production laufen?

Migration 031b_bootstrap_super_admin.sql legt automatisch einen super_admin aus ENV-Variablen an. Auf Production-Server muss INITIAL_SUPER_ADMIN_PASSWORD in .env.prod stehen — kurz, danach gelöscht/rotiert werden. Alternative: nur das interaktive npm run bootstrap:super-admin nutzen, 031b reine Skeleton-Migration. Empfehlung tendenziell: Bootstrap rauslassen aus Production-Migration, nur interaktiv — sicherer. Bestätigung?

❓ #F3 — Wird ein zweiter Server (Staging) gewünscht oder reicht der IONOS-VPS als „Production = Dev"?

Aktueller Plan: 1 Server (IONOS VPS 4-4-120). Sobald erster zahlender Kunde da ist, riskant — jede Code-Änderung geht live. Variante mit Staging: 2. IONOS-Server, ähnliche Größe, ~6 €/Monat extra. GitHub Actions deployt automatisch nach Push, Promotion zu Production manuell.
